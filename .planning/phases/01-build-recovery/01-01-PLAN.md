<objective>
Create the normalization foundation: a pure, idempotent `normalizeRecordMap()` that
unwraps the double-wrapped Notion recordMap (`{ value: { value, role } }` → `{ role, value }`),
and a single `getNotionPage(id)` infra entry point that composes `notionAPI.getPage` with it.

Purpose: react-notion-x expects flat `{ role, value }` records. The unofficial `notion-client`
now returns double-wrapped records, causing `uuidToId(undefined).replaceAll(...)` to crash the
whole SSG build. This plan builds the one transform + one entry point every consumer will route
through (BUILD-01, D-01/D-02/D-03). It creates the contract; the consumers are wired in Plan 02.

Output: `normalize-record-map.ts` (new) and an extended `notion-client.ts` exporting `getNotionPage`.
</objective>
